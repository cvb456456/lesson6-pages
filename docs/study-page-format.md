# 日语学习页格式规范

新增课程页按这个规范执行，避免后续页面格式漂移。

## 必须接入的位置

- `LearnJapan/middle-study-index.html`：新增课程入口和完成状态。
- 所有既有课程页的 `course-switch`：补上新页面链接。
- `lesson6-pages/sync-from-source.ps1`：补上源页面、目标页面、封面图复制项。

## 歌曲精读页

- 文件名使用 `song-*.html`，仍加入 `middle-study-index.html` 和所有课程页 `course-switch`。
- 歌曲页可以把四个学习视图命名为 `全曲地图 / 逐句歌词 / 语法词汇 / 练习复习`。
- 每句歌词必须有日文、平假名读法、中文意思、整句结构、语法重点、按语法断句、重点词汇。
- 可以不用课程截图，但必须有明确的音乐主题视觉，并保证手机端逐句卡片可读。
- 全曲听力精讲沿用 `AudioLessonPlayer.mount`，每句至少包含日语原句、中文意思、结构说明、语法重点、词汇搭配。

## 逐句精读结构

逐句精读统一使用第6课样式：

- `details.sentence` 折叠卡片。
- `summary` 中显示编号、说话人或段落类型、日语原句、小喇叭。
- 展开后必须有：
  - `听整句精讲`。
  - 中文翻译。
  - 整句读法。
  - 句子逻辑。
  - `按语法断句` 的 flex 词块卡片。
  - 语法与阅读提示。
  - 重点词块。
  - 可复用结构或词块解析。

不要在逐句精读里使用三列表格；手机端会拥挤，且不利于点击播放。

## 移动端要求

- 620px 以下隐藏 `展开` 字样，只保留编号和主体。
- 断句词块允许换行，单个词块可占满一行。
- 日语长句、公司名、邮箱必须 `overflow-wrap:anywhere`。
- 页面不得出现横向滚动。

## 音频要求

- 原句、语法块、词汇块都应有 `data-speech` 小喇叭。
- 整句精讲必须接入 `AudioLessonPlayer.mount`。
- 改动 `data-speech` 或 `data-narration-*` 后必须重新运行 `generate-audio.ps1`。

## 练习要求

- 必须保留选择、排序、填空三类题。
- 选择题选项随机化，正确答案不能固定在第一项。
- 排序和填空判定应忽略常见标点和空格。

## 发布检查

1. `node --check` 检查内联脚本。
2. Edge headless 渲染页面，检查 `details.sentence`、`class="syntax"`、音频 manifest 和封面图。
3. 运行 `generate-audio.ps1`。
4. 运行 `sync-from-source.ps1`。
5. 提交、推送。
6. 验证线上页面和至少一个音频文件返回 `200`。
